# Telegram Shop Bot

Дипломная работа: "Разработка Telegram-бота для автоматизации процесса продажи товаров с функционалом оформления заказов и интеграцией платёжных систем"

## Архитектура

- **Backend**: NestJS + PostgreSQL + TypeORM
- **Telegram Bot**: NestJS Telegraf (полная интеграция с БД)
- **Client Site**: React + Vite + TypeScript
- **Admin Panel**: React + Vite + TypeScript
- **Payments**: ЮKassa, Stripe, СБП

## Структура проекта

```
telegram-shop/
├── backend/          # NestJS API + Telegram Bot
├── client-site/      # React клиентский сайт
├── admin-panel/      # React админ-панель
└── docker-compose.yml
```

## 🤖 Telegram Bot - Главная фича!

### Полноценный магазин в Telegram:
- ✅ Автоматическая регистрация по Telegram ID
- ✅ Просмотр каталога с фотографиями
- ✅ Категории товаров
- ✅ Корзина (всегда видна в меню)
- ✅ Оформление заказов
- ✅ История заказов
- ✅ Синхронизация с веб-сайтом

### Быстрый старт бота:
```bash
# 1. Создайте бота через @BotFather в Telegram
# 2. Добавьте токен в backend/.env:
TELEGRAM_BOT_TOKEN=ваш_токен

# 3. Запустите backend
cd backend
npm run start:dev

# 4. Найдите бота в Telegram и нажмите /start
```

## Функционал

- Аналитика и отчёты (графики продаж)

- Логи системы

## Установка

### Быстрый старт (Docker)

```bash
# Запустить всё сразу
docker-compose up -d

# Backend: http://localhost:3000
# Client Site: http://localhost:3001
# Admin Panel: http://localhost:5173
```

### Разработка (локально)

```bash
# 1. База данных
docker-compose up -d postgres

# 2. Backend + Telegram Bot
cd backend
npm install
cp .env.example .env
# Добавьте TELEGRAM_BOT_TOKEN в .env
npm run start:dev

# 3. Client Site
cd client-site
npm install
npm run dev

# 4. Admin Panel
cd admin-panel
npm install
npm run dev
```

## 🔑 Ключевая особенность: Единая система

### Как это работает:

1. **Пользователь запускает бота** → автоматически создается в БД
2. **Добавляет товары в корзину** → сохраняется в общей БД
3. **Может продолжить на сайте** → та же корзина, те же заказы
4. **Или оформить заказ в боте** → заказ виден везде

### Схема интеграции:

```
Telegram Bot ─┐
              ├─→ PostgreSQL ←─┬─ Client Site
Admin Panel ──┘                └─ Admin Panel
```

Все работают с одной базой данных!

## 🎯 Примеры использования

### Сценарий 1: Покупка через бота
```
1. Пользователь запускает бота в Telegram
2. Выбирает "🛍 Каталог"
3. Нажимает "➕ 2 шт" на товаре
4. Переходит в "🛒 Корзину"
5. Нажимает "✅ Оформить заказ"
6. Получает номер заказа
7. Оплачивает на сайте или с баланса
```

### Сценарий 2: Начал в боте, закончил на сайте
```
1. Добавил товары в корзину через бота
2. Перешел на сайт
3. Увидел те же товары в корзине
4. Оформил и оплатил заказ
5. Получил уведомление в боте
```

### Сценарий 3: Админ обрабатывает заказ
```
1. Заказ создан (в боте или на сайте)
2. Админ видит его в админ-панели
3. Меняет статус на "Обработка"
4. Клиент видит обновление в боте и на сайте
```
