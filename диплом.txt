Аннотация

Рыбачук А.В. Разработка Telegram-бота для автоматизации процесса продажи товаров с функционалом оформления заказов и интеграцией платёжных систем

В работе проведен анализ современных решений автоматизации онлайн-продаж через мессенджеры, изучены возможности платформы Telegram и Telegram Bot API. Разработана модульная архитектура программной системы, включающая серверную часть на базе фреймворка NestJS, базу данных PostgreSQL, Telegram-бота, клиентский веб-сайт и административную панель. Реализован функционал каталога товаров с фильтрацией по категориям, система корзины с синхронизацией между интерфейсами, механизм оформления заказов. Выполнена интеграция с платежной системой ЮKassa для онлайн-оплаты, реализована система внутренних кошельков пользователей.

Ключевые слова: Telegram-бот, электронная коммерция, автоматизация продаж, платежные системы, NestJS, React, PostgreSQL, интернет-магазин.


Abstract

Rybachuk A.V. Development of a Telegram bot for automating the sales process with order processing functionality and payment system integration

The thesis analyzes modern solutions for automating online sales through messengers and explores the capabilities of the Telegram platform and Telegram Bot API. A modular architecture of the software system was developed, including a server part based on the NestJS framework, PostgreSQL database, Telegram bot, client website, and administrative panel. The functionality of the product catalog with category filtering, cart system with interface synchronization, and order processing mechanism were implemented. Integration with the YooKassa payment system for online payments and an internal user wallet system were realized.

Keywords: Telegram bot, e-commerce, sales automation, payment systems, NestJS, React, PostgreSQL, online store.

Актуальность темы.  В условиях стремительного развития электронной коммерции и цифровых технологий автоматизация процессов продаж становится одной из ключевых задач для современных компаний и индивидуальных предпринимателей. Все больше пользователей совершают покупки через интернет, при этом важную роль играет удобство взаимодействия с клиентом, скорость обработки заказов и простота оформления покупки. Одним из эффективных инструментов для организации онлайн-продаж являются мессенджеры, которые активно используются миллионами пользователей по всему миру.
Платформа Telegram предоставляет широкие возможности для разработки программных решений в виде ботов, способных автоматизировать различные бизнес-процессы. Использование Telegram-ботов позволяет реализовать удобный интерфейс для клиентов, автоматизировать процесс выбора товаров, оформления заказов и осуществления оплаты через интеграцию с платежными системами. Это значительно упрощает взаимодействие между продавцом и покупателем, снижает нагрузку на персонал и повышает эффективность работы бизнеса.
Таким образом, разработка Telegram-бота для автоматизации процесса продажи товаров с функционалом оформления заказов и интеграцией платежных систем является актуальной задачей, направленной на повышение эффективности электронных продаж и улучшение пользовательского опыта.

ГЛАВА 1. ТЕОРЕТИЧЕСКИЕ АСПЕКТЫ РАЗРАБОТКИ TELEGRAM-БОТА ДЛЯ АВТОМАТИЗАЦИИ ПРОЦЕССА ПРОДАЖИ ТОВАРОВ С ФУНКЦИОНАЛОМ ОФОРМЛЕНИЯ ЗАКАЗОВ И ИНТЕГРАЦИЕЙ ПЛАТЁЖНЫХ СИСТЕМ
1.1. Особенности использования Telegram-ботов для автоматизации процесса продажи товаров
Развитие информационных технологий и распространение интернет-коммуникаций существенно изменили способы взаимодействия бизнеса с клиентами. В последние годы особую популярность получили мессенджеры, которые используются не только для личного общения, но и для решения различных задач в сфере электронной коммерции. Одним из наиболее востребованных мессенджеров является Telegram, который предоставляет широкие возможности для разработки автоматизированных сервисов, в том числе Telegram-ботов.
Telegram-бот представляет собой программное приложение, функционирующее внутри мессенджера Telegram и взаимодействующее с пользователями посредством текстовых сообщений, команд, кнопок и других элементов интерфейса. Боты позволяют автоматизировать различные процессы, обеспечивая выполнение определённых функций без непосредственного участия человека. Благодаря использованию Telegram Bot API разработчики могут создавать ботов, способных выполнять широкий спектр задач, включая обработку запросов пользователей, предоставление информации, сбор данных и интеграцию с внешними сервисами.
Одной из наиболее востребованных областей применения Telegram-ботов является автоматизация процессов электронной торговли. Использование ботов позволяет организовать процесс продажи товаров непосредственно в интерфейсе мессенджера, что значительно упрощает взаимодействие между продавцом и покупателем. Пользователь получает возможность ознакомиться с ассортиментом товаров, выбрать необходимую продукцию, оформить заказ и произвести оплату, не покидая приложения Telegram.
Одним из ключевых преимуществ Telegram-ботов является удобство использования. Пользователям не требуется устанавливать дополнительные приложения или переходить на сторонние сайты, так как все необходимые действия выполняются в привычной среде мессенджера. Это снижает барьер входа для клиентов и способствует увеличению количества совершаемых покупок. Кроме того, Telegram отличается высокой скоростью работы, стабильностью соединения и поддержкой различных платформ, включая мобильные устройства, планшеты и персональные компьютеры.
Еще одним важным преимуществом использования Telegram-ботов является возможность автоматизации большого количества рутинных операций. Бот может самостоятельно отвечать на запросы пользователей, предоставлять информацию о товарах, принимать заказы, фиксировать контактные данные клиентов и отправлять уведомления о статусе заказа. Это позволяет значительно сократить нагрузку на сотрудников компании и повысить эффективность работы бизнеса. Автоматизация таких процессов особенно актуальна для интернет-магазинов и малого бизнеса, где ресурсы для обслуживания клиентов могут быть ограничены.
В процессе реализации системы онлайн-продаж с использованием Telegram-бота важную роль играет организация удобного пользовательского интерфейса. Несмотря на то, что взаимодействие с ботом происходит в формате диалога, Telegram предоставляет различные инструменты для построения интерактивного интерфейса. К таким инструментам относятся клавиатуры с кнопками, инлайн-кнопки, меню выбора, а также возможность отправки мультимедийных сообщений, содержащих изображения товаров, описания и стоимость. Использование данных элементов позволяет реализовать структуру каталога товаров, систему навигации и пошаговый процесс оформления заказа.
Одной из ключевых функций Telegram-бота, используемого в электронной коммерции, является возможность оформления заказов. В рамках данного процесса пользователь выбирает товар из каталога, указывает необходимые параметры (например, количество товара), вводит контактные данные и подтверждает заказ. После этого информация о заказе сохраняется в базе данных и может быть обработана системой или передана администратору для дальнейшей обработки. В некоторых случаях также реализуется возможность отслеживания статуса заказа и получения уведомлений о его изменении.
Не менее важным аспектом является интеграция Telegram-бота с платежными системами. Современные системы электронной коммерции предполагают возможность онлайн-оплаты товаров с использованием различных методов, таких как банковские карты, электронные кошельки и другие цифровые способы оплаты. Telegram предоставляет встроенный механизм работы с платежами, позволяющий подключать внешние платежные сервисы и осуществлять обработку транзакций непосредственно в интерфейсе мессенджера. Благодаря этому пользователи могут оплачивать заказы быстро и безопасно, не переходя на сторонние ресурсы.
При разработке Telegram-бота для автоматизации продаж важную роль играет также организация хранения и обработки данных. Информация о пользователях, товарах и заказах должна сохраняться в базе данных, что позволяет обеспечивать корректную работу системы, формировать статистику продаж и управлять ассортиментом продукции. Для реализации таких задач обычно используется серверная часть приложения, которая обрабатывает запросы от Telegram-бота, взаимодействует с базой данных и обеспечивает интеграцию с внешними сервисами.
В ряде случаев Telegram-бот используется не как полностью самостоятельное решение, а как часть более крупной программной системы. В частности, бот может работать совместно с веб-сайтом, который выполняет дополнительные функции. Например, через веб-интерфейс может осуществляться управление каталогом товаров, редактирование информации о продукции, обработка заказов и просмотр статистики продаж. Такой подход позволяет разделить пользовательский интерфейс для клиентов и административную часть системы, обеспечивая более удобное управление интернет-магазином.
Использование веб-сайта совместно с Telegram-ботом также позволяет реализовать дополнительные возможности для пользователей. Например, сайт может предоставлять расширенный каталог товаров, подробные описания продукции, систему авторизации или личный кабинет клиента. При этом Telegram-бот выполняет роль удобного канала коммуникации с пользователями и инструмента для быстрого оформления заказов. Интеграция этих компонентов позволяет создать комплексную систему электронной коммерции, сочетающую преимущества мессенджеров и веб-технологий.
Еще одной важной особенностью Telegram-ботов является возможность интеграции с различными внешними сервисами и программными системами. Боты могут взаимодействовать с системами управления контентом, базами данных, CRM-системами, службами доставки и платежными платформами. Благодаря использованию программных интерфейсов (API) становится возможным создание гибких и масштабируемых решений, способных адаптироваться к потребностям бизнеса.
Таким образом, Telegram-боты являются эффективным инструментом для автоматизации процесса продажи товаров и взаимодействия с клиентами. Они обеспечивают удобный и быстрый способ оформления заказов, позволяют интегрировать платежные системы и автоматизировать обработку данных. В сочетании с веб-сайтом Telegram-бот может стать частью комплексной информационной системы, предназначенной для управления онлайн-продажами и повышения эффективности коммерческой деятельности. Использование подобных решений позволяет компаниям оптимизировать бизнес-процессы, улучшить качество обслуживания клиентов и повысить конкурентоспособность на рынке электронной коммерции.
1.2. Анализ существующих программных решений
Развитие электронной коммерции и популярность мессенджеров привели к появлению большого количества программных решений, предназначенных для автоматизации продаж товаров через чат-ботов. Такие системы позволяют предпринимателям организовывать взаимодействие с клиентами, управлять каталогом товаров, принимать заказы и осуществлять оплату. На современном рынке существуют как специализированные платформы для создания Telegram-ботов, так и комплексные сервисы автоматизации продаж, включающие интеграцию с различными каналами коммуникации и платежными системами.
Анализ существующих решений позволяет определить их основные возможности, выявить преимущества и недостатки, а также определить требования к разрабатываемой системе.
Одним из известных сервисов автоматизации продаж через мессенджеры является платформа ManyChat. Данный сервис предназначен для создания чат-ботов без необходимости программирования и используется для автоматизации взаимодействия с клиентами. Платформа поддерживает интеграцию с несколькими каналами связи, включая мессенджеры и социальные сети. С помощью конструктора пользователь может создавать сценарии диалогов, настраивать автоматические ответы и реализовывать элементы каталога товаров. Также система поддерживает интеграцию с внешними сервисами и CRM-системами.
Однако возможности ManyChat в отношении Telegram ограничены по сравнению с другими платформами, так как основное внимание сервиса направлено на автоматизацию взаимодействия в социальных сетях. Кроме того, функциональность настройки сложных сценариев продаж и интеграции платежных систем может быть ограничена без использования дополнительных инструментов. 1.4. Выбор технологического стека и платформы разработки
При разработке программной системы для автоматизации продажи товаров через Telegram-бота и веб-интерфейс важную роль играет правильный выбор технологического стека. От используемых технологий зависит производительность системы, её масштабируемость, безопасность, удобство поддержки и возможность дальнейшего развития. В рамках данной работы была разработана многокомпонентная система, включающая серверную часть, Telegram-бота, административную панель управления, клиентский веб-сайт и инфраструктурные компоненты.
Архитектура системы построена по принципу разделения на несколько независимых компонентов: backend-сервер, Telegram-бот, административный интерфейс и клиентский сайт. Такой подход позволяет обеспечить гибкость разработки, упростить поддержку системы и обеспечить возможность масштабирования отдельных частей приложения.
Серверная часть системы
Серверная часть системы реализована на базе фреймворка NestJS, который является современным фреймворком для разработки серверных приложений на платформе Node.js. NestJS использует архитектурный подход, основанный на принципах модульности и внедрения зависимостей, что позволяет создавать масштабируемые и поддерживаемые программные системы.
Использование NestJS позволяет структурировать приложение на отдельные модули, каждый из которых отвечает за определенную функциональность системы. Например, отдельные модули могут отвечать за работу с пользователями, управление товарами, обработку заказов, взаимодействие с платежными системами и работу Telegram-бота. Такой подход облегчает разработку и тестирование системы, а также делает код более понятным и поддерживаемым.
В качестве основного языка программирования используется TypeScript. TypeScript представляет собой надмножество языка JavaScript, которое добавляет статическую типизацию и расширенные возможности разработки. Использование TypeScript позволяет обнаруживать ошибки на этапе компиляции, улучшает читаемость кода и повышает надежность программного обеспечения. Благодаря строгой типизации разработчик получает возможность более эффективно управлять сложными структурами данных и логикой приложения.
Для работы с базой данных используется библиотека TypeORM, которая представляет собой объектно-реляционный маппер (ORM). ORM позволяет взаимодействовать с базой данных через объектную модель, что значительно упрощает разработку. Вместо написания сложных SQL-запросов разработчик может работать с сущностями и репозиториями, описанными в виде классов. TypeORM поддерживает различные типы баз данных и предоставляет инструменты для управления миграциями, что позволяет изменять структуру базы данных в процессе развития проекта.
В качестве системы управления базами данных используется PostgreSQL. PostgreSQL является одной из наиболее популярных реляционных баз данных с открытым исходным кодом. Она обеспечивает высокую надежность, производительность и поддержку сложных транзакций. PostgreSQL широко применяется в современных веб-приложениях благодаря своей устойчивости к большим нагрузкам и поддержке расширенных возможностей работы с данными.
Для реализации системы аутентификации пользователей применяется библиотека Passport, которая является популярным инструментом для организации механизмов аутентификации в приложениях на Node.js. В сочетании с использованием технологии JWT (JSON Web Token) реализуется безопасная система авторизации пользователей. JWT позволяет передавать информацию о пользователе между клиентом и сервером в виде подписанного токена, что обеспечивает безопасное взаимодействие без необходимости постоянного хранения сессий на сервере.
Для обеспечения безопасности хранения пользовательских паролей используется библиотека Bcrypt. Данная библиотека предназначена для криптографического хеширования паролей. Вместо хранения пароля в открытом виде система сохраняет его хешированное значение, что значительно повышает уровень безопасности данных пользователей и снижает риск их компрометации.
Для взаимодействия с внешними сервисами и API используется библиотека Axios, представляющая собой HTTP-клиент для выполнения сетевых запросов. Axios используется для отправки запросов к платежным системам, внешним сервисам и другим компонентам системы. Библиотека поддерживает удобную работу с асинхронными запросами и обработку ответов сервера.
Для реализации загрузки файлов в системе применяется библиотека Multer, предназначенная для обработки multipart-запросов. Данный инструмент используется для загрузки изображений товаров и других файлов, которые могут использоваться в системе интернет-магазина.
Для тестирования серверной части используется фреймворк Jest. Jest предоставляет инструменты для написания модульных и интеграционных тестов, что позволяет проверять корректность работы отдельных компонентов системы и снижать вероятность возникновения ошибок при внесении изменений в код.
Реализация Telegram-бота
Telegram-бот является одним из основных компонентов разработанной системы и служит интерфейсом взаимодействия пользователей с интернет-магазином. Для реализации бота используется библиотека Telegraf, которая является одной из наиболее популярных библиотек для работы с Telegram Bot API в среде Node.js.
Telegraf предоставляет удобный программный интерфейс для обработки сообщений пользователей, создания интерактивных кнопок, меню, форм и других элементов взаимодействия. С помощью данной библиотеки реализуется логика работы бота, включая просмотр каталога товаров, оформление заказов, получение уведомлений и взаимодействие с платежными системами.
Интеграция Telegram-бота с серверной частью приложения осуществляется через API, реализованное на базе NestJS. Такой подход позволяет отделить бизнес-логику системы от интерфейса взаимодействия с пользователем и обеспечивает возможность расширения функциональности в будущем.
Административная панель управления
Для управления системой разработана административная панель, предназначенная для работы администратора интернет-магазина. Через данный интерфейс осуществляется управление товарами, обработка заказов, просмотр информации о пользователях и управление содержимым системы.
Административная панель реализована с использованием библиотеки React, которая является одной из наиболее популярных JavaScript-библиотек для создания пользовательских интерфейсов. React позволяет строить интерфейсы на основе компонентов, что обеспечивает удобство разработки и повторное использование элементов интерфейса.
Для разработки административной панели также используется язык TypeScript, который обеспечивает типизацию и повышает надежность кода.
Сборка и запуск проекта осуществляется с помощью инструмента Vite, который является современным сборщиком веб-приложений. Vite обеспечивает высокую скорость сборки проекта и удобную среду разработки благодаря использованию современных механизмов модульной загрузки.
Для организации навигации между страницами используется библиотека React Router DOM, которая предоставляет инструменты для реализации клиентской маршрутизации в приложениях React.
Для управления состоянием данных и выполнения запросов к серверу используется библиотека TanStack Query (React Query). Данный инструмент обеспечивает удобную работу с асинхронными запросами, автоматическое кеширование данных и обновление информации при изменении состояния системы.
Для выполнения HTTP-запросов к серверной части приложения используется библиотека Axios, которая позволяет удобно взаимодействовать с REST API.
Клиентский веб-сайт
Помимо Telegram-бота система включает клиентский веб-сайт, предназначенный для взаимодействия с пользователями через браузер. Сайт позволяет просматривать каталог товаров, получать подробную информацию о продукции и оформлять заказы.
Клиентский сайт также реализован с использованием библиотеки React и языка TypeScript. Для сборки проекта используется инструмент Vite, обеспечивающий быструю разработку и оптимизацию производительности приложения.
Навигация между страницами сайта реализована с помощью библиотеки React Router DOM, а управление состоянием данных и выполнение сетевых запросов осуществляется с использованием TanStack Query и Axios.
Инфраструктура системы
Для развертывания и управления компонентами системы используется технология контейнеризации Docker. Контейнеризация позволяет упаковать приложение вместе со всеми зависимостями в изолированную среду, что обеспечивает одинаковую работу системы на различных серверах и упрощает процесс развертывания.
Для управления несколькими контейнерами используется инструмент Docker Compose, который позволяет описать инфраструктуру системы в виде конфигурационного файла. С помощью Docker Compose запускаются контейнеры серверной части, базы данных и веб-сервера.
В качестве веб-сервера для обслуживания клиентских приложений используется Nginx. Данный сервер обеспечивает обработку HTTP-запросов, раздачу статических файлов и маршрутизацию сетевого трафика между компонентами системы.
База данных PostgreSQL также запускается в контейнере Docker с использованием облегченной версии образа PostgreSQL Alpine, что позволяет снизить потребление ресурсов сервера.
Интеграция платежных систем
Одной из ключевых функций системы является возможность онлайн-оплаты заказов. Для реализации данной возможности была реализована интеграция с несколькими платежными системами.
В системе используется платежный сервис ЮKassa (YooKassa), который является популярным платежным агрегатором и позволяет принимать оплату банковскими картами, электронными кошельками и другими способами оплаты.
Также поддерживается интеграция с международной платежной системой Stripe, которая широко используется для обработки онлайн-платежей и обеспечивает высокий уровень безопасности транзакций.
Дополнительно реализована поддержка Системы быстрых платежей (СБП), которая позволяет осуществлять оплату через банковские приложения путем сканирования QR-кода или перехода по платежной ссылке.
Использование нескольких платежных систем позволяет повысить гибкость системы и предоставить пользователям различные способы оплаты заказов.
Таким образом, выбранный технологический стек обеспечивает создание современной, масштабируемой и надежной системы автоматизации продажи товаров. Использование современных фреймворков, библиотек и инфраструктурных решений позволяет обеспечить высокую производительность системы, удобство разработки и возможность дальнейшего расширения функциональности.

Использование Telegram-ботов позволяет автоматизировать значительную часть процессов, связанных с обработкой запросов пользователей, предоставлением информации о товарах и оформлением заказов. Благодаря удобному интерфейсу и высокой доступности мессенджеров пользователи могут взаимодействовать с системой без необходимости установки дополнительных приложений или перехода на сторонние ресурсы.
Также были рассмотрены особенности реализации Telegram-ботов в системах электронной коммерции. Было установлено, что такие боты могут выполнять функции каталога товаров, обеспечивать навигацию по ассортименту, принимать заказы и передавать информацию о них в серверную часть системы. Кроме того, Telegram предоставляет возможности для интеграции с платежными сервисами, что позволяет реализовать полный цикл онлайн-продаж непосредственно в интерфейсе мессенджера.
В рамках анализа существующих программных решений были рассмотрены популярные платформы и сервисы, предназначенные для создания чат-ботов и автоматизации взаимодействия с пользователями. К таким решениям относятся различные конструкторы чат-ботов и платформы электронной коммерции. Проведенный анализ показал, что существующие сервисы обладают рядом преимуществ, включая простоту использования и быстрое развертывание. Однако многие из них имеют ограничения, связанные с гибкостью настройки бизнес-логики, возможностями интеграции с внешними системами и масштабируемостью.
В результате анализа было установлено, что для реализации системы автоматизации продаж целесообразно разрабатывать собственное программное решение, которое позволит более гибко реализовать необходимые функции, включая управление каталогом товаров, обработку заказов и интеграцию с платежными системами.
Также в первой главе был обоснован выбор технологического стека для реализации системы. Были рассмотрены современные технологии разработки серверной части, пользовательских интерфейсов и инфраструктуры развертывания. В качестве основных технологий были выбраны фреймворк NestJS для реализации серверной логики, библиотека Telegraf для разработки Telegram-бота, база данных PostgreSQL для хранения информации, а также библиотека React для разработки административной панели и клиентского веб-сайта.
Кроме того, была рассмотрена инфраструктура развертывания системы с использованием технологий контейнеризации Docker и веб-сервера Nginx. Такой подход позволяет обеспечить удобство развертывания, масштабируемость и независимость компонентов системы. Также была предусмотрена интеграция с несколькими платежными системами, что обеспечивает возможность удобной и безопасной оплаты заказов пользователями.
Таким образом, проведенное исследование позволило определить основные требования к разрабатываемой системе и выбрать технологические решения, наиболее подходящие для реализации поставленных задач. Полученные результаты теоретического анализа служат основой для разработки программной системы автоматизации продажи товаров, которая будет рассмотрена во второй главе выпускной квалификационной работы.

2.1 

ГЛАВА 2. РАЗРАБОТКА TELEGRAM-БОТА ДЛЯ АВТОМАТИЗАЦИИ ПРОЦЕССА ПРОДАЖИ ТОВАРОВ С ФУНКЦИОНАЛОМ ОФОРМЛЕНИЯ ЗАКАЗОВ И ИНТЕГРАЦИЕЙ ПЛАТЁЖНЫХ СИСТЕМ

2.1. Проектирование программной системы автоматизации продаж

На основе проведенного в первой главе анализа существующих решений и технологий была разработана концепция программной системы для автоматизации процесса продажи товаров с использованием Telegram-бота и веб-интерфейса. Проектирование системы включало определение функциональных требований, разработку сценариев использования и формирование общей логики работы приложения.

Основной целью разрабатываемой системы является создание удобного и эффективного инструмента для организации онлайн-продаж товаров, который позволит пользователям осуществлять покупки через мессенджер Telegram или веб-сайт, а администраторам предоставит возможность управления каталогом товаров и обработки заказов. Система должна обеспечивать автоматизацию основных бизнес-процессов, связанных с продажей товаров, включая просмотр каталога, добавление товаров в корзину, оформление заказов и осуществление оплаты.

При формировании требований к системе были определены основные группы пользователей и их потребности. Первую группу составляют покупатели, которые используют систему для поиска и приобретения товаров. Данная категория пользователей должна иметь возможность просматривать каталог товаров, получать подробную информацию о продукции, добавлять товары в корзину, оформлять заказы и производить оплату. Вторую группу составляют администраторы системы, которые отвечают за управление содержимым интернет-магазина. Администраторы должны иметь возможность добавлять новые товары, редактировать информацию о существующей продукции, управлять категориями товаров, просматривать и обрабатывать заказы, а также получать статистическую информацию о работе системы.

Функциональные требования к системе были сформулированы на основе анализа потребностей пользователей и особенностей процесса онлайн-продаж. Система должна обеспечивать следующие основные функции. Во-первых, управление каталогом товаров, включая возможность добавления, редактирования и удаления товаров, организацию товаров по категориям, загрузку изображений продукции и управление информацией о наличии товаров на складе. Во-вторых, обработка заказов, включая возможность добавления товаров в корзину, оформления заказа с указанием контактных данных и адреса доставки, автоматического уменьшения количества товара на складе при оформлении заказа и отслеживания статуса заказа. В-третьих, интеграция с платежными системами для обеспечения возможности онлайн-оплаты заказов различными способами. В-четвертых, управление пользователями, включая регистрацию и авторизацию пользователей, хранение информации о клиентах и их заказах. В-пятых, административные функции для управления системой, включая просмотр статистики продаж, управление заказами и обработку информации о пользователях.

Нефункциональные требования к системе включают обеспечение высокой производительности и отзывчивости интерфейса, надежности работы системы и сохранности данных, безопасности хранения персональной информации пользователей и платежных данных, масштабируемости системы для обработки растущего количества пользователей и заказов, а также удобства использования интерфейса как для покупателей, так и для администраторов.

Для описания взаимодействия пользователей с системой были разработаны сценарии использования. Основные сценарии включают следующие процессы. Первый сценарий описывает процесс просмотра каталога товаров. Пользователь запускает Telegram-бота или открывает веб-сайт, выбирает интересующую категорию товаров, просматривает список доступных товаров с изображениями и описаниями, получает подробную информацию о выбранном товаре, включая цену и наличие на складе. Второй сценарий описывает добавление товара в корзину. Пользователь выбирает товар из каталога, указывает необходимое количество единиц товара, добавляет товар в корзину, система сохраняет информацию о выбранных товарах и отображает общее количество товаров в корзине. Третий сценарий описывает оформление заказа. Пользователь переходит в корзину и просматривает выбранные товары, проверяет правильность выбора и при необходимости изменяет количество товаров или удаляет ненужные позиции, нажимает кнопку оформления заказа, вводит контактные данные и адрес доставки, выбирает способ оплаты, подтверждает заказ, система создает заказ в базе данных и уменьшает количество товара на складе, пользователь получает подтверждение о создании заказа с указанием номера заказа. Четвертый сценарий описывает оплату заказа. Пользователь выбирает способ оплаты из доступных вариантов, система перенаправляет пользователя на страницу платежной системы или предоставляет инструкции по оплате, пользователь вводит платежные данные и подтверждает оплату, платежная система обрабатывает транзакцию и отправляет уведомление о результате оплаты, система обновляет статус заказа в соответствии с результатом оплаты. Пятый сценарий описывает управление товарами администратором. Администратор входит в административную панель, выбирает раздел управления товарами, добавляет новый товар с указанием названия, описания, цены, количества на складе и категории, загружает изображение товара, сохраняет информацию о товаре в базе данных, при необходимости редактирует или удаляет существующие товары.

Логика работы системы построена на взаимодействии нескольких компонентов. Telegram-бот служит основным интерфейсом для взаимодействия пользователей с системой через мессенджер. Бот обрабатывает команды пользователей, отображает информацию о товарах, принимает заказы и передает данные на серверную часть приложения. Серверная часть реализует бизнес-логику системы, обрабатывает запросы от Telegram-бота и веб-интерфейсов, взаимодействует с базой данных для хранения и получения информации, обеспечивает интеграцию с платежными системами и управляет процессом обработки заказов. База данных хранит всю информацию о пользователях, товарах, заказах, транзакциях и других данных системы. Веб-интерфейсы включают клиентский сайт для покупателей и административную панель для управления системой. Платежные системы обеспечивают обработку онлайн-платежей и передачу информации о результатах транзакций обратно в систему.

Важным аспектом проектирования системы является обеспечение синхронизации данных между различными компонентами. Пользователь должен иметь возможность начать оформление заказа в Telegram-боте, а затем продолжить процесс на веб-сайте, при этом информация о выбранных товарах должна сохраняться. Для реализации такой функциональности все компоненты системы работают с единой базой данных, что обеспечивает согласованность информации независимо от того, через какой интерфейс пользователь взаимодействует с системой.

Также при проектировании системы было уделено внимание вопросам безопасности. Пароли пользователей должны храниться в зашифрованном виде с использованием криптографических алгоритмов хеширования. Для авторизации пользователей используется механизм токенов, что позволяет обеспечить безопасное взаимодействие между клиентской и серверной частями приложения. Платежные данные обрабатываются через защищенные каналы связи с использованием протокола HTTPS, а сама обработка платежей осуществляется на стороне платежных систем, что снижает риски компрометации финансовой информации.

Таким образом, на этапе проектирования были определены основные требования к системе, разработаны сценарии использования и сформирована общая логика работы приложения. Полученные результаты служат основой для разработки архитектуры системы и реализации отдельных компонентов программного решения.

2.2. Разработка архитектуры и структуры системы

На основе сформулированных требований и сценариев использования была разработана архитектура программной системы автоматизации продажи товаров. Архитектура системы построена по принципу разделения на несколько независимых компонентов, каждый из которых выполняет определенные функции и взаимодействует с другими компонентами через программные интерфейсы.

Общая структура системы включает следующие основные компоненты. Первым компонентом является серверная часть приложения, которая реализует бизнес-логику системы и обеспечивает обработку запросов от клиентских приложений. Серверная часть разработана с использованием фреймворка NestJS и представляет собой REST API, через который осуществляется взаимодействие всех компонентов системы. Вторым компонентом является база данных PostgreSQL, которая хранит всю информацию о пользователях, товарах, заказах, транзакциях и других данных системы. Третьим компонентом является Telegram-бот, который служит интерфейсом для взаимодействия пользователей с системой через мессенджер Telegram. Четвертым компонентом является клиентский веб-сайт, который предоставляет пользователям возможность просматривать каталог товаров и оформлять заказы через браузер. Пятым компонентом является административная панель, которая предназначена для управления системой администраторами. Шестым компонентом является веб-сервер Nginx, который обеспечивает обработку HTTP-запросов, раздачу статических файлов и маршрутизацию трафика между компонентами системы.

Взаимодействие между компонентами системы организовано следующим образом. Telegram-бот получает сообщения от пользователей через Telegram Bot API, обрабатывает команды и формирует запросы к серверной части приложения. Серверная часть получает запросы от Telegram-бота и веб-интерфейсов, обрабатывает их в соответствии с бизнес-логикой, взаимодействует с базой данных для получения или сохранения информации и возвращает результаты обработки клиентским приложениям. База данных хранит всю информацию системы и обеспечивает быстрый доступ к данным через SQL-запросы. Клиентский веб-сайт и административная панель взаимодействуют с серверной частью через REST API, отправляя HTTP-запросы и получая ответы в формате JSON. Веб-сервер Nginx принимает входящие HTTP-запросы, направляет их к соответствующим компонентам системы и обеспечивает раздачу статических файлов клиентских приложений.

Серверная часть приложения организована по модульному принципу. Каждый модуль отвечает за определенную функциональность системы и включает контроллеры для обработки HTTP-запросов, сервисы для реализации бизнес-логики и репозитории для взаимодействия с базой данных. Основные модули серверной части включают модуль аутентификации, который обеспечивает регистрацию и авторизацию пользователей, генерацию и проверку токенов доступа, модуль управления пользователями, который предоставляет функции для работы с информацией о пользователях, модуль управления товарами, который реализует функции добавления, редактирования, удаления и получения информации о товарах, модуль управления категориями, который обеспечивает организацию товаров по категориям, модуль корзины, который управляет добавлением товаров в корзину и их удалением, модуль заказов, который обрабатывает создание заказов, изменение их статусов и получение информации о заказах, модуль платежей, который обеспечивает интеграцию с платежными системами и обработку транзакций, модуль Telegram-бота, который реализует логику работы бота и обработку команд пользователей.

Структура базы данных разработана с учетом требований к хранению информации о всех объектах системы. Основные таблицы базы данных включают таблицу пользователей, которая хранит информацию о зарегистрированных пользователях системы, включая идентификатор Telegram, электронную почту, хешированный пароль, имя, фамилию, телефон и другие данные, таблицу товаров, которая содержит информацию о товарах, включая название, описание, цену, количество на складе, ссылку на изображение, идентификатор категории и статус активности, таблицу категорий, которая хранит информацию о категориях товаров, таблицу корзины, которая содержит информацию о товарах, добавленных пользователями в корзину, таблицу заказов, которая хранит информацию о созданных заказах, включая номер заказа, идентификатор пользователя, общую сумму, статус заказа, способ оплаты, адрес доставки и контактный телефон, таблицу позиций заказа, которая содержит информацию о товарах, входящих в состав каждого заказа, таблицу кошельков, которая хранит информацию о внутренних кошельках пользователей, включая текущий баланс, общую сумму пополнений и общую сумму расходов, таблицу транзакций, которая содержит информацию о всех финансовых операциях пользователей, включая пополнения баланса, оплату заказов и возвраты средств.

Между таблицами базы данных установлены связи, обеспечивающие целостность данных. Каждый товар связан с категорией через внешний ключ. Каждая позиция в корзине связана с пользователем и товаром. Каждый заказ связан с пользователем, а позиции заказа связаны с заказом и товаром. Каждая транзакция связана с пользователем и может быть связана с заказом. Использование внешних ключей и каскадных операций обеспечивает автоматическое удаление связанных записей при удалении основных объектов.

Архитектура клиентских приложений построена на основе компонентного подхода с использованием библиотеки React. Каждый компонент представляет собой независимую часть пользовательского интерфейса, которая может быть повторно использована в различных частях приложения. Основные компоненты клиентского веб-сайта включают компонент каталога товаров, который отображает список доступных товаров с возможностью фильтрации по категориям и поиска, компонент карточки товара, который показывает подробную информацию о выбранном товаре, компонент корзины, который отображает список товаров, добавленных в корзину, с возможностью изменения количества и удаления товаров, компонент оформления заказа, который обеспечивает ввод контактных данных, выбор адреса доставки и способа оплаты, компонент списка заказов, который показывает историю заказов пользователя с информацией о статусах, компонент профиля пользователя, который позволяет просматривать и редактировать личную информацию, компонент финансов, который отображает информацию о балансе кошелька, истории транзакций и предоставляет возможность пополнения баланса.

Административная панель включает компоненты для управления товарами, категориями, заказами и пользователями. Компонент управления товарами предоставляет возможность просмотра списка товаров, добавления новых товаров с загрузкой изображений, редактирования существующих товаров и удаления товаров. Компонент управления категориями позволяет создавать, редактировать и удалять категории товаров. Компонент управления заказами отображает список всех заказов с возможностью просмотра подробной информации и изменения статусов заказов. Компонент статистики показывает общую информацию о количестве товаров, категорий, заказов и пользователей в системе.

Telegram-бот реализован с использованием библиотеки Telegraf и интегрирован с серверной частью приложения. Архитектура бота включает обработчики команд, которые реагируют на текстовые команды пользователей, обработчики кнопок, которые обрабатывают нажатия на инлайн-кнопки и кнопки клавиатуры, обработчики callback-запросов, которые обрабатывают действия пользователей с интерактивными элементами, функции формирования сообщений, которые создают текстовые сообщения с информацией о товарах, заказах и других данных, функции взаимодействия с API, которые отправляют запросы к серверной части для получения или сохранения информации.

Для обеспечения надежности и масштабируемости системы используется технология контейнеризации Docker. Каждый компонент системы упакован в отдельный контейнер, что обеспечивает изоляцию процессов и упрощает развертывание. Контейнеры управляются с помощью Docker Compose, который позволяет описать всю инфраструктуру системы в конфигурационном файле и запустить все компоненты одной командой. Использование контейнеров также обеспечивает одинаковую работу системы на различных серверах и упрощает процесс обновления отдельных компонентов.

Веб-сервер Nginx выполняет роль обратного прокси-сервера и обеспечивает маршрутизацию запросов между компонентами системы. Запросы к API направляются на серверную часть приложения, а запросы к статическим файлам обрабатываются непосредственно Nginx. Такая архитектура позволяет эффективно распределять нагрузку и обеспечивает высокую производительность системы.

Таким образом, разработанная архитектура системы обеспечивает четкое разделение функциональности между компонентами, удобство разработки и поддержки, возможность масштабирования отдельных частей системы и высокую надежность работы. Модульная структура серверной части и компонентный подход в клиентских приложениях позволяют легко добавлять новую функциональность и вносить изменения в существующий код без влияния на другие части системы.


2.3. Реализация Telegram-бота и серверной части приложения

После разработки архитектуры системы был реализован программный код основных компонентов приложения. Реализация началась с разработки серверной части и Telegram-бота, так как эти компоненты составляют основу всей системы и обеспечивают обработку бизнес-логики.

Серверная часть приложения была реализована с использованием фреймворка NestJS на языке программирования TypeScript. Первым этапом разработки стало создание структуры проекта и настройка основных модулей. Был создан главный модуль приложения, который импортирует все остальные модули системы и настраивает подключение к базе данных. Для работы с базой данных PostgreSQL используется библиотека TypeORM, которая обеспечивает объектно-реляционное отображение и позволяет работать с данными через объектную модель.

Для каждой таблицы базы данных была создана соответствующая сущность в виде класса TypeScript. Сущности описывают структуру данных и связи между таблицами. Например, сущность товара содержит поля для хранения названия, описания, цены, количества на складе, ссылки на изображение и идентификатора категории. Сущность заказа включает поля для хранения номера заказа, идентификатора пользователя, общей суммы, статуса заказа, способа оплаты и адреса доставки. Использование декораторов TypeORM позволяет описать структуру таблиц и связи между ними непосредственно в коде, что упрощает разработку и обеспечивает автоматическую синхронизацию структуры базы данных с моделью данных приложения.

Модуль аутентификации реализует функции регистрации и авторизации пользователей. При регистрации нового пользователя система проверяет уникальность электронной почты, хеширует пароль с использованием библиотеки Bcrypt и сохраняет информацию о пользователе в базе данных. При авторизации система проверяет правильность введенного пароля путем сравнения его хеша с сохраненным значением и в случае успешной проверки генерирует токен доступа с использованием технологии JWT. Токен содержит идентификатор пользователя и информацию о его роли в системе. Клиентское приложение сохраняет полученный токен и отправляет его в заголовках всех последующих запросов к серверу для подтверждения личности пользователя.

Модуль управления товарами предоставляет функции для работы с каталогом продукции. Реализованы методы для получения списка всех товаров с возможностью фильтрации по категориям, получения подробной информации о конкретном товаре, добавления нового товара с загрузкой изображения, редактирования существующего товара и удаления товара из каталога. При добавлении товара администратор указывает название, описание, цену, количество на складе и выбирает категорию. Изображение товара загружается на сервер с использованием библиотеки Multer, которая обрабатывает multipart-запросы и сохраняет файлы в указанной директории. Путь к сохраненному изображению записывается в базу данных вместе с остальной информацией о товаре.

Модуль корзины обеспечивает управление товарами, добавленными пользователями для последующей покупки. Реализованы методы для добавления товара в корзину с указанием количества, изменения количества товара в корзине, удаления товара из корзины и получения списка всех товаров в корзине пользователя. При добавлении товара в корзину система проверяет наличие достаточного количества товара на складе. Если товар уже находится в корзине пользователя, система увеличивает его количество, в противном случае создается новая запись в таблице корзины. Информация о корзине привязана к конкретному пользователю через его идентификатор, что позволяет каждому пользователю иметь свою независимую корзину.

Модуль заказов реализует функции создания и обработки заказов. При оформлении заказа система получает от пользователя информацию о товарах в корзине, контактные данные и адрес доставки. Система проверяет наличие всех товаров на складе в необходимом количестве. Если проверка пройдена успешно, создается новый заказ с уникальным номером, который формируется на основе текущей даты и порядкового номера заказа. Для каждого товара в заказе создается отдельная запись в таблице позиций заказа, которая содержит информацию о товаре, его количестве и цене на момент оформления заказа. После создания заказа система автоматически уменьшает количество товаров на складе и увеличивает счетчик продаж для каждого товара. Товары удаляются из корзины пользователя. Заказу присваивается начальный статус, который может быть изменен администратором в процессе обработки.

Модуль платежей обеспечивает интеграцию с платежными системами и управление внутренним кошельком пользователей. Для каждого пользователя автоматически создается внутренний кошелек, на который можно пополнять средства и использовать их для оплаты заказов. Реализованы функции для получения информации о балансе кошелька, пополнения баланса через платежные системы, оплаты заказа с использованием средств кошелька и получения истории всех транзакций пользователя. При пополнении баланса система создает платеж в платежной системе ЮKassa и получает ссылку для перенаправления пользователя на страницу оплаты. После успешной оплаты платежная система отправляет уведомление на сервер, и система автоматически пополняет баланс кошелька пользователя. При оплате заказа с использованием кошелька система проверяет достаточность средств на балансе, списывает необходимую сумму и создает запись о транзакции.

Telegram-бот был реализован с использованием библиотеки Telegraf, которая предоставляет удобный интерфейс для работы с Telegram Bot API. Бот интегрирован с серверной частью приложения и использует те же модули для работы с данными. При запуске бота регистрируются обработчики различных команд и событий. Основные команды включают команду старт, которая приветствует пользователя и отображает главное меню, команду каталог, которая показывает список категорий товаров, команду корзина, которая отображает содержимое корзины пользователя, команду заказы, которая показывает историю заказов, команду помощь, которая предоставляет информацию о возможностях бота.

При первом обращении пользователя к боту система автоматически создает учетную запись пользователя в базе данных, используя информацию из профиля Telegram, включая идентификатор пользователя, имя пользователя, имя и фамилию. Это позволяет пользователям начать работу с системой без необходимости регистрации. Идентификатор Telegram используется для связи учетной записи пользователя с его действиями в боте.

Главное меню бота реализовано с использованием клавиатуры с кнопками, которая постоянно отображается внизу экрана. Меню включает кнопки для доступа к основным функциям системы, таким как просмотр категорий товаров, переход в корзину, просмотр заказов, вход на веб-сайт и получение помощи. Кнопка корзины отображает текущее количество товаров в корзине, что позволяет пользователю всегда видеть состояние своей корзины.

При выборе категории товаров бот отображает список доступных категорий с указанием количества товаров в каждой категории. Категории представлены в виде инлайн-кнопок, при нажатии на которые бот показывает список товаров выбранной категории. Для каждого товара отображается изображение, название, описание, цена и количество на складе. Под каждым товаром размещены кнопки для быстрого добавления товара в корзину в количестве одной, двух или трех единиц. Также присутствует кнопка для перехода в корзину. После добавления товара в корзину бот обновляет кнопки под товаром, показывая текущее количество данного товара в корзине пользователя.

Корзина отображает список всех товаров, добавленных пользователем, с указанием количества и общей стоимости каждой позиции. В конце списка показывается общая сумма заказа. Под списком товаров размещены кнопки для оформления заказа и очистки корзины. При нажатии кнопки оформления заказа бот запрашивает у пользователя номер телефона для связи. После ввода телефона бот запрашивает адрес доставки. Когда вся необходимая информация получена, бот создает заказ в системе и отправляет пользователю подтверждение с номером заказа и информацией о сумме. Товары автоматически удаляются из корзины, а количество товаров на складе уменьшается.

Раздел заказов показывает историю всех заказов пользователя, разделенную на активные и завершенные заказы. Для каждого заказа отображается номер, дата создания, сумма и текущий статус. Пользователь может обновить информацию о статусах заказов, нажав соответствующую кнопку.

Для обеспечения возможности входа на веб-сайт из Telegram-бота была реализована система авторизации через одноразовые токены. При нажатии кнопки входа на сайт бот генерирует специальный токен, который действителен в течение десяти минут и может быть использован только один раз. Токен сохраняется в базе данных вместе с идентификатором пользователя Telegram и временем истечения. Бот отправляет пользователю ссылку, содержащую этот токен. При переходе по ссылке веб-сайт проверяет токен, получает информацию о пользователе и автоматически авторизует его в системе, выдавая обычный токен доступа JWT. После использования токен удаляется из базы данных, что предотвращает его повторное использование.

Также в боте реализован раздел помощи, который содержит ответы на часто задаваемые вопросы. Пользователь может выбрать интересующую тему из списка, и бот отобразит подробную информацию по выбранному вопросу. Темы включают инструкции по оформлению заказа, информацию о способах оплаты, условиях доставки, правилах возврата товаров, управлении аккаунтом и контактных данных службы поддержки.

Для обеспечения надежной работы бота была реализована обработка ошибок. Если при выполнении операции возникает ошибка, бот отправляет пользователю понятное сообщение о проблеме и предлагает повторить действие или обратиться в службу поддержки. Все ошибки логируются на сервере для последующего анализа и устранения проблем.

Серверная часть приложения также включает систему логирования, которая записывает информацию о всех важных событиях, включая запросы к API, ошибки обработки, изменения в базе данных и действия пользователей. Логи помогают отслеживать работу системы, выявлять проблемы и анализировать поведение пользователей.

Для обеспечения безопасности API были реализованы механизмы защиты от несанкционированного доступа. Все запросы к защищенным эндпоинтам требуют наличия действительного токена доступа в заголовке запроса. Система проверяет токен, извлекает из него информацию о пользователе и проверяет его права доступа к запрашиваемому ресурсу. Администраторы имеют расширенные права доступа и могут выполнять операции управления системой, недоступные обычным пользователям.

Также была настроена политика CORS, которая определяет, какие домены могут отправлять запросы к API. Это предотвращает несанкционированный доступ к серверу со сторонних сайтов и повышает безопасность системы.

Таким образом, реализация серверной части и Telegram-бота обеспечила создание функциональной системы, способной обрабатывать запросы пользователей, управлять данными и обеспечивать взаимодействие между различными компонентами приложения. Использование современных технологий и библиотек позволило создать надежный и производительный программный продукт.

2.4. Реализация функционала оформления заказов и интеграции платёжных систем

Одной из ключевых функций разработанной системы является возможность оформления заказов и осуществления оплаты через различные платежные системы. Реализация данного функционала потребовала разработки нескольких взаимосвязанных компонентов, обеспечивающих полный цикл обработки заказа от момента добавления товаров в корзину до получения подтверждения об оплате.

Процесс оформления заказа начинается с добавления товаров в корзину. Пользователь может добавлять товары как через Telegram-бота, так и через веб-сайт. Информация о корзине хранится в базе данных и привязана к учетной записи пользователя, что обеспечивает синхронизацию корзины между различными интерфейсами. Пользователь может в любой момент просмотреть содержимое корзины, изменить количество товаров или удалить ненужные позиции.

Когда пользователь готов оформить заказ, он переходит на страницу оформления заказа в веб-интерфейсе или нажимает соответствующую кнопку в Telegram-боте. На этом этапе система проверяет наличие всех товаров в корзине на складе. Если какого-либо товара недостаточно, пользователь получает уведомление о необходимости уменьшить количество или удалить товар из корзины. Эта проверка предотвращает создание заказов на товары, которых нет в наличии.

После успешной проверки наличия товаров пользователю предлагается указать контактные данные и адрес доставки. В веб-интерфейсе пользователь может выбрать один из ранее сохраненных адресов или добавить новый адрес. Адреса сохраняются в профиле пользователя и могут быть использованы при оформлении последующих заказов. В Telegram-боте пользователь вводит номер телефона и адрес доставки в текстовом виде. Система сохраняет эту информацию вместе с заказом.

Следующим этапом является выбор способа оплаты. Система предоставляет несколько вариантов оплаты заказа. Первый вариант - оплата с использованием внутреннего кошелька. Каждый пользователь имеет внутренний кошелек, на который можно пополнять средства и использовать их для оплаты заказов. При выборе этого способа система проверяет достаточность средств на балансе кошелька. Если средств достаточно, заказ создается и сразу помечается как оплаченный, а необходимая сумма списывается с кошелька пользователя. Второй вариант - оплата при получении. В этом случае заказ создается со статусом ожидания оплаты, и пользователь оплачивает товар при получении. Третий вариант - онлайн-оплата через платежные системы. Пользователь может выбрать оплату банковской картой или через систему быстрых платежей.

Для реализации онлайн-оплаты была выполнена интеграция с платежной системой ЮKassa. ЮKassa является популярным платежным агрегатором в России и поддерживает различные способы оплаты, включая банковские карты, электронные кошельки и систему быстрых платежей. Интеграция осуществляется через REST API платежной системы.

При выборе онлайн-оплаты система создает платеж в ЮKassa, отправляя запрос с информацией о сумме платежа, описании и адресе возврата пользователя после завершения оплаты. В запросе также передается уникальный идентификатор транзакции, который используется для связи платежа с заказом в системе. ЮKassa возвращает ссылку на страницу оплаты, на которую перенаправляется пользователь. На странице оплаты пользователь вводит данные банковской карты или выбирает другой способ оплаты и подтверждает платеж.

После обработки платежа ЮKassa отправляет уведомление на специальный адрес сервера, который называется webhook. Webhook представляет собой эндпоинт API, который принимает POST-запросы от платежной системы с информацией о статусе платежа. При получении уведомления система проверяет статус платежа. Если платеж успешно завершен, система находит соответствующую транзакцию в базе данных по идентификатору платежа, обновляет статус транзакции на завершенную и пополняет баланс кошелька пользователя на сумму платежа. Если платеж был отменен или завершился с ошибкой, статус транзакции обновляется соответствующим образом.

Для обеспечения безопасности взаимодействия с платежной системой используется аутентификация с помощью идентификатора магазина и секретного ключа, которые передаются в заголовках HTTP-запросов. Также используется механизм идемпотентности, который предотвращает дублирование платежей при повторной отправке запроса. Каждый запрос к API ЮKassa содержит уникальный ключ идемпотентности, который формируется на основе идентификатора транзакции и текущего времени.

Помимо пополнения баланса кошелька, пользователи могут использовать средства кошелька для оплаты заказов. При оформлении заказа с оплатой через кошелек система проверяет баланс, создает заказ, списывает необходимую сумму с кошелька и создает запись о транзакции типа оплата заказа. Эта транзакция связана с заказом через идентификатор заказа, что позволяет отслеживать, на что были потрачены средства.

Также реализована возможность возврата средств на кошелек в случае отмены заказа. Если администратор отменяет заказ, который был оплачен с использованием кошелька, система автоматически возвращает средства на баланс пользователя и создает транзакцию типа возврат.

Для удобства пользователей была разработана страница финансов в веб-интерфейсе, которая предоставляет полную информацию о состоянии кошелька и истории транзакций. Страница разделена на несколько вкладок. Первая вкладка - обзор, которая показывает текущий баланс кошелька, общую сумму пополнений, общую сумму расходов и количество транзакций. Также отображаются последние пять транзакций с информацией о типе, сумме, статусе и дате. Вторая вкладка - история транзакций, которая показывает полный список всех транзакций пользователя в виде таблицы с возможностью фильтрации по типу и статусу. Третья вкладка - пополнение баланса, которая предоставляет интерфейс для пополнения кошелька. Пользователь может выбрать способ оплаты, ввести сумму пополнения или выбрать одну из предложенных сумм и нажать кнопку пополнения. После этого пользователь перенаправляется на страницу оплаты платежной системы.

При возврате пользователя на сайт после оплаты система автоматически проверяет статус последнего платежа и обновляет баланс кошелька, если платеж был успешно завершен. Пользователь видит уведомление об успешном пополнении баланса.

Все транзакции хранятся в базе данных с подробной информацией о типе операции, сумме, статусе, способе оплаты, описании и связанных объектах. Это позволяет вести полный учет финансовых операций пользователей и предоставлять им детальную информацию о движении средств на кошельке.

Для обеспечения целостности данных при работе с финансовыми операциями используются транзакции базы данных. Операции списания и пополнения средств выполняются атомарно, что предотвращает возможность потери или дублирования средств в случае сбоев системы.

Также была реализована система уведомлений о статусе заказов. Когда администратор изменяет статус заказа в административной панели, система может отправлять уведомления пользователю через Telegram-бота. Это позволяет пользователям оперативно получать информацию о ходе обработки их заказов.

Таким образом, реализация функционала оформления заказов и интеграции платежных систем обеспечила создание полноценной системы электронной коммерции, которая позволяет пользователям удобно совершать покупки и оплачивать заказы различными способами. Интеграция с платежной системой ЮKassa обеспечивает безопасную обработку онлайн-платежей, а внутренний кошелек предоставляет пользователям дополнительную гибкость в управлении своими средствами.
