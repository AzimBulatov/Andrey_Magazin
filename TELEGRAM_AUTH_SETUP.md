# Настройка авторизации через Telegram

## Описание

Реализована система авторизации пользователей Telegram на веб-сайте через одноразовые magic links.

## Как это работает

1. Пользователь в Telegram боте нажимает кнопку "🌐 Войти на сайт" или вводит команду `/website`
2. Бот генерирует одноразовый токен (действует 10 минут)
3. Бот отправляет ссылку с токеном пользователю
4. Пользователь переходит по ссылке
5. Сайт проверяет токен и автоматически авторизует пользователя
6. Токен удаляется после использования

## Безопасность

- Токен действителен только 10 минут
- Токен одноразовый (после использования становится недействительным)
- Токен привязан к конкретному Telegram ID
- Старые неиспользованные токены автоматически удаляются

## Настройка

### Backend

1. Добавьте в `.env` переменную `WEBSITE_URL`:
```env
WEBSITE_URL="http://localhost:5173"  # для разработки
# или
WEBSITE_URL="https://yourdomain.com"  # для продакшена
```

2. Таблица `telegram_auth_tokens` создастся автоматически при запуске приложения

### Frontend

Роут `/auth/telegram` уже настроен и обрабатывает авторизацию

## API Endpoints

### POST /auth/telegram/generate-token
Генерирует токен для авторизации

**Body:**
```json
{
  "telegramId": "123456789"
}
```

**Response:**
```json
{
  "token": "abc123..."
}
```

### POST /auth/telegram/validate-token
Валидирует токен и возвращает JWT

**Body:**
```json
{
  "token": "abc123..."
}
```

**Response:**
```json
{
  "access_token": "jwt_token...",
  "user": {
    "id": 1,
    "telegramId": "123456789",
    "username": "user",
    "firstName": "John",
    "lastName": "Doe",
    "email": null,
    "phone": null,
    "role": "user"
  }
}
```

## Тестирование

1. Запустите backend: `cd backend && npm run start:dev`
2. Запустите frontend: `cd client-site && npm run dev`
3. Откройте Telegram бот
4. Нажмите "🌐 Войти на сайт"
5. Перейдите по ссылке
6. Вы должны быть автоматически авторизованы на сайте

## Возможные ошибки

### "Токен не найден"
- Проверьте, что токен передается в URL параметре `?token=...`

### "Токен недействителен или истек"
- Токен действителен только 10 минут
- Токен можно использовать только один раз
- Запросите новую ссылку в боте

### "Пользователь не найден"
- Убедитесь, что пользователь существует в базе данных
- Пользователь должен хотя бы раз взаимодействовать с ботом

## Продакшен

Для продакшена:
1. Измените `WEBSITE_URL` на реальный домен
2. Убедитесь, что используется HTTPS
3. Настройте CORS для вашего домена
4. Установите `synchronize: false` в TypeORM и используйте миграции
