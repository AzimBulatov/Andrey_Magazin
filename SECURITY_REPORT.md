# Отчет по безопасности

## ✅ ЗАЩИЩЕНО

### 1. SQL-инъекции
- **Статус:** Полностью защищено
- **Детали:** Все SQL запросы используют параметризованные запросы через TypeORM и pg с плейсхолдерами ($1, $2)
- **Примеры:**
  ```javascript
  await client.query('DELETE FROM admins WHERE email = $1', [email]);
  await client.query('SELECT id FROM users WHERE email = $1', [email]);
  ```

### 2. Открытые порты
- **Статус:** Защищено
- **Открыты только:**
  - 22 (SSH) - защищен fail2ban + сильный пароль
  - 80 (HTTP) - nginx с rate limiting
- **Закрыты:**
  - 5432 (PostgreSQL) - доступен только внутри Docker сети
  - 6379 (Redis) - доступен только внутри Docker сети
  - 3000 (Backend) - доступен только через nginx

### 3. Rate Limiting (DDoS защита)
- **Статус:** Настроено
- **Лимиты:**
  - API endpoints: 30 запросов/сек (burst 50)
  - Auth endpoints: 5 попыток/минуту (защита от брутфорса)
  - Общий лимит: 100 запросов/сек
  - Соединения с IP: максимум 20 одновременно

### 4. Секреты
- **Статус:** Защищено
- `.env` файлы в `.gitignore` - не попадают в репозиторий
- Пароли хешируются через bcrypt

### 5. Автоматические бэкапы
- **Статус:** Работает
- Ежедневно в 3:00 AM
- Хранятся 7 дней
- Папка: `/root/backups`

### 6. SSH
- **Статус:** Защищено
- Fail2ban блокирует после нескольких неудачных попыток
- Сильный пароль (20+ символов)

## ⚠️ НИЗКИЙ РИСК

### 1. NPM уязвимости
- **Статус:** Не критично
- 11 уязвимостей в dev-зависимостях (@nestjs/cli, @nestjs/schematics)
- Не работают в продакшене (только при разработке)
- Уязвимости: ReDoS в ajv, path-to-regexp, picomatch

### 2. Устаревшие пакеты
- **Рекомендация:** Периодически обновлять зависимости
- **Команда:** `docker exec telegram-shop-backend npm update`
